Github Page - 使用 Cloudflare 購買網域並套用至你的網站 (2022.7.5 版本)

發表於2022-07-05|更新於2022-07-05|Domain

Github Page 網址

我們使用 Github Page 預設的網址會像這樣：<username>.github.io，
雖然有基本的功能，連線速度不錯，有免費的 SSL Cert，但是你不能使用自訂網域。
所以接下來我會教大家如何購買一個你專屬的網域並且套用至你的網站。

Prerequisite

  • 信用卡一張 (付錢買網域)
  • Cloudflare 帳號 (Free)
  • 已有 github page (<username>.github.io)

Cloudflare

Cloudflare 是一間美國 CDN 服務的一個大型企業，能提供專業的防 DDoS 服務，加強你網站的瀏覽速度等，其總部位於加利福尼亞州舊金山。

首頁

在開始之前，請先登入你的 Cloudflare 帳號，如果沒有的話，請先申請一個。

登入後會看到這個頁面。

購買網域 - 第一步

點擊左方的 購買網域 (新的) 然後再點擊 註冊網域 (新的)。

購買網域 - 第二步 (尋找網域名稱)

現在會看到 Find a domain，請輸入你想要的名稱，這個會作為你的網站名稱 ~
輸入完成確定想要這個名稱後請按下 搜索 ~

購買網域 - 第三步 (尋找想要的 TLDs)

此時會列出你輸入的名稱所存在的 TLDS 列表，列如 .com .net .io 等等，
但要注意每一個 TLDs 價錢也不一樣，就好像 .com 就是最便宜的 TLDs。

選擇一個你喜歡的就好。(有錢的話不用煩)

購買網域 - 第四步 (輸入註冊者資訊)

輸入你申請此網域的個人資訊。

購買網域 - 第四步 (輸入信用卡號碼)

輸入你付款的信用卡資料。

購買網域 - 第五步 (網域購買完成)

完成輸入後確定送出付款。
在付款的時候需要時間處理加密傳送 (大概 20秒)。

購買網域 - 已完成購買

然後在首頁就會看到你剛剛購買的網域已經新增在主頁了 ~


首頁 (Cloudflare)

當我們購買完成後，我們還需要設置 DNS 相關的操作。

修改 DNS 設置 - 第一步

回到首頁後按下剛購買的網域。

修改 DNS 設置 - 第二步

點擊 DNS。

修改 DNS 設置 - 第三步

我們需要新增五條規則，如下圖。

  • 新增四個 A Records，
    名稱為 @ (你申請的網域)，
    內容為 Github 官方文件提供 的四個 IPV4 位置 185.199.108-111.153。
  • 新增一個 CNAME Record，
    名稱為 www，
    內容為你的 github page 網址 <username>.github.io。

修改 DNS 設置 - 第三步 (可選)

這邊是新增 IPV6 的規則，可選擇做還是不做，如果做的話請跟以下位置：

  • 新增四個 AAAA Records，
    名稱為 @ (你申請的網域)，
    內容為 Github 官方文件提供 提供的四個 IPV6 位置 2606:50c0:8000-8003::153

Githuhb repo - 修改 DNS

現在我們需要新增你 repo 內的自訂網域，
登入你的 github，然後回到你的 repo。

修改 DNS 設置 - 第一步 (開啟 Settings)

按下你 repo 內的 Settings。

修改 DNS 設置 - 第二步 (開啟 Pages)

按下 Setting 內的 Pages。

修改 DNS 設置 - 第三步 (Custom domain)

於 Custom domain 輸入你購買的網域連結，然後按下 Save。
此時 Github Page 會進行 DNS Check in Progress 請耐心等待 ~

修改 DNS 設置 - 新增完成 (顯示完成)

等待完成後會出現 DNS check successful。

修改 DNS 設置 - 新增完成 (等待)

輸入完成後回到你的 repo 會發現多了一個檔案叫 CNAME，而內容是你的網域。

完成 設置

此事此刻你輸入網域後就會發現已經成功，而你 <username>.github.io 也會重定向到你的網域。

Hexo CNAME

如果你是使用 Hexo 生成網頁，一定要在 source 目錄底下建立一個檔案，名為 CNAME，然後內容輸入你的網域。

這樣做的原因是為了讓 Hexo 每次 Push 上 github 的時候都會將你存在的檔案送上去，然後如果你本身檔案內沒有 CNAME 的話，那麼你再次上傳檔案的時候，因為你本機沒有 CNAME，此時網域就沒有了。所以這個步驟相當重要！
