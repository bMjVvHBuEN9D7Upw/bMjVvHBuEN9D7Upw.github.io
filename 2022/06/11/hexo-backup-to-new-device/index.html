Hexo - 備份至新電腦繼續寫文章

發表於2022-06-11|更新於2022-06-11|NPMHexo

警告

備份是應該每天要做的事情，
如果操作失敗而檔案沒有了，
我本人也沒有能力為大家解決。

Hexo 架構 (butterfly)

紫色筆是不用備份的資料。

  • .deploy_git (不用備份)
    這個資料夾是你在 github page repo內的內容，
    會出現是因為使用了 hexo g 生產，
    並將生產內容上傳至 repo。
  • .github (不用備份)
    不重要，可以無視 haha
  • node_modules
    NPM 套件，這個可以在之後安裝，
    輸入 npm install 安裝。
    由於檔案大，可以刪除。
  • scaffolds (請備份)
    這個是你的網站 layout 的呼叫。
  • source (超重要，請備份)
    這個是你網站內所有的文章和圖片，
    請整個備份。
  • themes (請備份)
    是你所用的主題，
    請整個備份。
  • .gitignore (請備份)
    每次 push 的時候會無視哪一個檔案或者資料夾。
  • _config.butterfly.yml (請備份)
    我的網站所用的主題 butterfly 的配置檔案 (yml)。
  • _config.yml (請備份)
    你網站的基本配置檔案 (yml)。
  • db.json (不用備份)
    不重要，這只是你每次使用 hexo server，
    開啟伺服器後生產的 cache 檔案。
    可以使用 hexo clean 清除。
  • package.json (重要，請備份)
    這個是你的網站內的所有所用的 NPM 套件，
    npm install 會根據這個檔案去進行下載。
  • package-lock.json (重要，請備份)
    這個是根據上面的 package.json 去進行指定版本下載，
    如果不使用，下載來的是最新版本 (包括未隱定的版本)，
    使用 -lock 就只會使用你你指定的 NPM 套件版本。
  • yarn.lock (重要，請備份)
    鎖定你的 yarn 套件版本。

開始

Step 1 :

git clone 你的網站 (等同於 .deploy_git 資料夾內容)

Step 2 :

開啟一個資料夾叫 public，
再將剛剛 clone 後的檔案放到 public 內

Step 3 :

除了 deploy_git, .github, node_modules, db.json 資料夾和檔案，
其他放到 clone 後的資料夾內。

Step 4 :

完成以上步驟後，開啟你的伺服器測試。
